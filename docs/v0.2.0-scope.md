# tui-components v0.2.0 — Scope reference

**Status:** In progress (dependent issues: REN-1319, REN-1330, REN-1331, REN-1332)
**Spec:** `rensei-architecture/014-tui-operator-surfaces.md`
**Umbrella issue:** REN-1318

This document is the canonical scope reference for the v0.2.0 milestone.
It lists all planned changes, the dependent issue that implements each, and the
architecture spec section each primitive maps to.

---

## Theme struct + swap mechanism (REN-1319)

Move from package-level palette vars to a swappable `Theme` struct.

- `theme.Theme` struct with all current palette fields.
- `theme.DefaultTheme()`, `theme.DarkTheme()`, `theme.HighContrastTheme()` constructors.
- `widget.WithTheme(t Theme)` option accepted by every widget.
- Per-tenant brand themes for SaaS consumers.

**Spec:** `014 § Theme swappability`
**Breaking change:** yes — see `MIGRATION-v0.2.0.md §1`.

---

## Capability registries (REN-1330)

Replace closed switch statements with open registry API.

- `ThemeRegistry` type with `RegisterStatus`, `RegisterWorkType`, `RegisterActivity`.
- `theme.Registry` package-level singleton.
- `theme.GetStatusStyle`, `theme.GetWorkTypeColor`, `theme.GetActivityIcon` delegate to registry.
- Fallback rendering for unknown kinds (symbol `"?"`, warning log).

**Spec:** `014 § Open registries`
**Breaking change:** behavioral (unknown kinds now log a warning instead of silently falling through).

## Format helpers + accessibility (REN-1332)

### New format helpers

| Helper | Description |
|---|---|
| `format.CapacityRatio(cur, max int)` | `"5 / 8"` or `"5 / ∞"` |
| `format.AttestationFingerprint(fp string)` | `"ed25519:abc1234…d4f2"` |
| `format.RegionList(regions []string, max int)` | `"iad1, +3 more"` |
| `format.ToolchainSpec(specs []string)` | `"java=17, node=20"` |
| `format.HumanLabel[T](val T, registry map[T]string)` | Typed-flag → human label |

### Accessibility

- `NO_COLOR` env var: symbol-first rendering, no ANSI colors.
- `RENSEI_A11Y=true` env var or `--a11y` flag: high-contrast + verbose labels.
- `AccessibleLabel string` field on every new primitive.
- Non-color disambiguators documented for every status/worktype/activity primitive.

**Spec:** `014 § Accessibility`
**Breaking change:** none — additive only.

## Release plan

1. REN-1319 (Theme) merges first — establishes `WithTheme` option on existing widgets.
2. REN-1330 (Registries) merges second — extends theme package with registry API.
3. REN-1331 (Primitives) merges third — adds new widgets consuming `Theme`.
4. REN-1332 (Format + A11y) merges fourth — additive only; no ordering constraint.
5. Tag `v0.2.0` on `main` once all four PRs are merged and CI is green.

See `MIGRATION-v0.2.0.md` for the consumer upgrade checklist.
